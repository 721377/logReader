Introduction

Welcome to the LogManager Dashboard documentation. This comprehensive guide will help you understand how to use all features of the platform.

The Logs Dashboard is a powerful tool for managing, visualizing, and analyzing application logs in real-time. It provides features for uploading JSON logs, filtering, searching, exporting, and viewing detailed log information with advanced analytics.

JSON Upload

Upload log files directly from your browser. No backend required - all processing happens client-side.

Soon JSON Upload Via API

Upload log files directly from the applications. using the LogManager APIS.

Advanced Filtering

Filter logs by user, company, level, date range, and search by event/details.

Export Data

Export filtered or all logs as JSON files for further analysis.

Real-time Stats

View statistics including total logs count and filtered logs count.

How to Use the Platform

1

Access the Dashboard

Open the Logs Dashboard in your browser. The interface shows a statistics header with total logs and filtered logs count, plus controls for filtering and searching.

2

Upload JSON Logs

Click the "Upload" section at the top. Select your JSON log file and click the "Upload" button. The logs will be processed instantly and added to your dashboard.

3

View Your Logs

Logs are displayed in a table grouped by date. Each row shows: Timestamp, User, Company, Event, Level, and a View button for details.

4

Filter & Search

Use the Filters section to narrow down logs by user name, company ID, log level, date range, or search in events/details.

5

View Details

Click the "View" button on any log row to see complete details including user action and additional data in a formatted table.

6

Export Logs

Click "Export" to download filtered logs (or all logs if no filter applied) as a JSON file.

Uploading JSON Files

Steps to Upload

  1. Prepare your JSON file with logs in the format specified below
  2. Go to the "Upload Logs" section at the top of the dashboard
  3. Click "Select JSON File" and choose your file
  4. Click the "Upload" button
  5. Wait for success message and logs will appear in the dashboard

‚úì Client-Side Processing: All uploads happen in your browser. No data is sent to any server. Logs are saved to your browser's local storage.

File Requirements

  • File must be valid JSON format (.json extension)
  • Can contain a single log object or an array of logs
  • Each log must have at least a timestamp field
  • Maximum file size depends on your browser's capabilities (typically several MB)

Upload Status Messages

After uploading, you'll see one of these messages:

Message Meaning
Successfully loaded X logs Upload successful! Logs are now in the dashboard.
Please select a file No file was selected. Click to choose a file first.
Please select a valid JSON file File is not JSON format. Make sure it has .json extension.
Error parsing JSON JSON syntax error. Check your file for valid JSON.

JSON Format Specification

Supported Log Structure

The platform expects logs with the following structure:

[ { "timestamp": "2025-11-18T11:55:52.441326Z", "date": "2025-11-18", "time": "12:55:52", "user": { "name": "john_doe" }, "agency": { "id": "18", "name": "Company Name" }, "event": "user_login_success", "details": "User successfully logged in", "level": "info", "userAction": "LOGIN", "additionalData": "{\"companyId\":\"18\",\"timestamp\":\"2025-11-18T12:55:58.446661\"}" } ]

Field Descriptions

Field Type Required Description
timestamp ISO 8601 String Required ISO 8601 formatted datetime (e.g., 2025-11-18T11:55:52.441326Z)
user Object Optional Object containing user info. Nested field: name (string)
agency Object Optional Object containing agency info. Nested fields: id (string), name (string)
event String Required Event type/name (e.g., "user_login_success")
details String Required Event details/description
level String Required Log level: "info", "warning", or "error"
userAction String Optional User action that triggered the log
additionalData String (JSON) or Object Optional Additional metadata. Can be JSON string or object. Displayed as table in modal.

Complete Example

[ { "timestamp": "2025-11-18T11:55:52.441326Z", "user": { "name": "Mauro.airgas" }, "agency": { "id": "18", "name": "4.0 Air Gas srl" }, "event": "ALLEGATI_CATEGORY_LOADED", "details": "Category and required allegati loaded", "level": "info", "userAction": "LOAD_ALLEGATI_PAGE", "additionalData": "{\"category\":\"CALDAIE\",\"requiredAllegati\":[\"ALL1\",\"RICEVUTA\"],\"idChiamata\":\"663428\"}" }, { "timestamp": "2025-11-18T11:56:05.132468Z", "user": { "name": "unknown_user" }, "agency": { "id": "unknown_company", "name": "unknown_agency" }, "event": "api_error", "details": "API request failed with status 500", "level": "error" } ]

Notes on Data Types

  • Timestamps: Must be ISO 8601 format for proper sorting and filtering
  • AdditionalData: If it's a JSON string, it will be parsed and displayed as a key-value table. If already an object, it will be used directly.
  • Log Level: Typically "info", "warning", or "error" - shown with colored badges
  • Fallbacks: If user/agency data is missing, system uses "unknown_user" and "unknown_company"

Filtering & Search Guide

Filter by User Name

Enter a partial or full username to filter logs. The search is case-insensitive and matches partial strings.

  • Example: type "john" to find logs from "john_doe", "johnny", etc.
  • Leave empty to show logs from all users

Filter by Company ID

Enter a company ID to filter logs from specific companies.

  • Example: type "18" to find all logs from company ID "18"
  • Exact match or partial match supported

Filter by Log Level

Select a log level from dropdown to filter:

  • info - Information logs
  • warning - Warning logs
  • error - Error logs

Filter by Date Range

Select "From" and "To" dates to filter logs within a specific date range.

  • Click date picker to select dates
  • Both dates are inclusive
  • Leave empty to ignore date filtering

Search in Event/Details

Search for keywords in event names or log details. Press Enter or click Search button.

  • Case-insensitive search
  • Searches both "event" and "details" fields
  • Example: search "login" to find all login-related events

Combine Multiple Filters

All filters work together to narrow down results:

Example: Filter by user "john" + company "18" + level "error" + date range = shows only error logs from john in company 18 during that date range

Clear All Filters

Click the "Clear" button to reset all filters and show all logs.
